# host-config

Host network configuration pipeline: Netbox-driven intent rendered to Netplan + cloud-init for B300 GPU hosts and CPU servers.

> **Status:** Tier 1 lab implementation in progress. See the [implementation plan][plan] for milestones and design intent.

[plan]: https://github.com/kg-aifabrik/research/blob/main/host-net-config/implementation-plan.md

## Overview

`host-config` produces cloud-init seeds for hosts from declarative intent stored in Netbox. The renderer is a FastAPI service; an nginx cache fronts it; cloud-init NoCloud on the target host pulls the seed at first boot. Soft-RoCE is used in Tier 1 for east-west NIC verbs validation.

See [docs/architecture/systems-overview.md](docs/architecture/systems-overview.md) for the component diagram and the canonical render flow.

## Prerequisites

- Python 3.12
- [uv](https://github.com/astral-sh/uv) for dependency management
- Docker (for Netbox container in dev)
- For lab operations: [`just`](https://github.com/casey/just), Ansible, KVM-enabled host (Lima on Mac, native on Linux, or a DigitalOcean Droplet)

## Quickstart

```bash
# Install deps and pre-commit hooks
uv sync
just hooks

# Run lint + tests
just lint
just test

# Bring up the lab (provision a DO Droplet + configure the full stack)
just lab-up         # provision + deploy-lab (Netbox, renderer, nginx, OVS, QEMU)
just lab-image      # sync tests/ (carries the e2e SSH key) + prepare the cloud image
just lab-test       # run the e2e suite on the Droplet
just lab-down       # tear down; verifies zero residual DO resources

# Or compose the whole thing: up → image → test → down, with trap-on-exit cleanup
just lab

# Iterate on a live Droplet after editing a template / renderer / fixture:
just lab-refresh    # rsync + restart renderer + flush nginx-cache + wait /healthz
```

> **Cold-start ordering matters:** `lab-image` runs after `lab-up` and
> before `lab-test` — it injects the e2e SSH key into the base image, and
> that key (under `tests/`) isn't synced by `lab-up`. `just lab` sequences
> these correctly. See [docs/runbooks/debug-cloud-init.md](docs/runbooks/debug-cloud-init.md).

## Configuration

Local secrets live in a gitignored `.env` file. Copy the template and fill in values:

```bash
cp .env.example .env
$EDITOR .env
```

Variables you'll need:

| Variable | Purpose | How to obtain |
|---|---|---|
| `NETBOX_API_TOKEN` | Authenticate against the local-dev Netbox | Generated by the `netbox-dev` Ansible role on first apply; see logs |
| `DIGITALOCEAN_TOKEN` | Provision DO Droplets for the lab | Create at https://cloud.digitalocean.com/account/api/tokens (read+write scope) |
| `SSH_KEY_FINGERPRINT` | Register your key with DO for SSH access | Run `doctl compute ssh-key list` after uploading your public key |

`just` recipes source `.env` automatically via `set dotenv-load`. No external tool (direnv etc.) required.
