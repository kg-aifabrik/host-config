# infra/ansible

Ansible roles and playbooks for the host-config lab.

## Layout

```
infra/ansible/
├── playbooks/        # composable entry points; one playbook per use-case
│   └── netbox-dev.yml
├── roles/            # role-per-component
│   └── netbox-dev/   # local Netbox via netbox-docker
└── inventory/        # generated/static inventories (DO Droplet inventory lands in M6)
```

## Quickstart (local Netbox)

```bash
# Requires: ansible (Homebrew: `brew install ansible`), Docker.
ansible-playbook -i localhost, -c local playbooks/netbox-dev.yml
```

After ~2 minutes the role prints a summary with the Netbox URL and the
path to the persisted API token. The token can be exported into the
project's `.env` as `NETBOX_API_TOKEN` for the renderer and fixture
loader to use.

## Linting

```bash
just lint-ansible   # runs ansible-lint over all roles and playbooks
```

## Roles

| Role | Purpose | Documented in |
|---|---|---|
| `netbox-dev` | Bring up local Netbox via netbox-docker | [roles/netbox-dev/README.md](roles/netbox-dev/README.md) |

Subsequent roles (`renderer`, `nginx-cache`, `ovs-harness`, `qemu-host`) land in M2–M5.

## Inventories

Local dev uses an inline inventory (`-i localhost,`).
Production / lab deployment uses `inventory/digitalocean.yml`, generated by `infra/ansible/playbooks/provision.yml` in M6-1.
